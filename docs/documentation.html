üè• System Overview

The Healthcare Face Recognition System is a comprehensive, privacy-preserving solution designed specifically for healthcare environments. It provides secure patient identification, AI-powered genetic screening, and bias detection capabilities.

Key Components

  • Patient Identification System with encrypted facial data storage
  • AI-powered genetic disorder pre-screening
  • Comprehensive bias detection and fairness monitoring
  • HIPAA/GDPR compliant consent management
  • Advanced anti-spoofing security measures
  • Real-time system monitoring and statistics

üì¶ Installation Guide

Prerequisites

  • Python 3.8 or higher
  • 4GB RAM minimum (8GB recommended)
  • Camera/webcam for face capture
  • Secure network environment

Quick Installation

# Clone the repository git clone https://github.com/omsherikar/face_recognition.git cd face_recognition # Run the installation script chmod +x install.sh ./install.sh # Start the server source venv/bin/activate python3 main.py
Note: The installation script will automatically create a virtual environment, install all dependencies, and set up the necessary directories.

üîå API Reference

POST /register-patient

Register a new patient with facial data and medical information.

Parameters

Parameter Type Required Description
patient_id string Yes Unique patient identifier
name string Yes Patient's full name
date_of_birth string Yes Patient's date of birth (YYYY-MM-DD)
medical_record_number string Yes Medical record number
consent_given boolean No Whether patient has given consent
image file Yes Patient's facial image (JPG, PNG, GIF)
POST /identify-patient

Identify a patient from facial image with optional genetic screening.

Parameters

Parameter Type Required Description
image file Yes Facial image for identification
session_id string No Session identifier for tracking
GET /system-statistics

Get comprehensive system statistics and monitoring data.

üîê Authentication

All API endpoints require authentication using Bearer tokens. Include the token in the Authorization header:

curl -H "Authorization: Bearer your_token_here" \ http://localhost:8001/system-statistics
Security Note: In production, use proper JWT tokens with expiration and validation. The current implementation uses placeholder authentication for demonstration purposes.

üí° Usage Examples

Register a Patient

curl -X POST http://localhost:8001/register-patient \ -H "Authorization: Bearer dummy_token" \ -F "patient_id=P001" \ -F "name=John Doe" \ -F "date_of_birth=1990-01-15" \ -F "medical_record_number=MRN123456" \ -F "consent_given=true" \ -F "image=@patient_photo.jpg"

Identify a Patient

curl -X POST http://localhost:8001/identify-patient \ -H "Authorization: Bearer dummy_token" \ -F "image=@patient_photo.jpg" \ -F "session_id=checkin_001"

Update Consent

curl -X POST http://localhost:8001/update-consent \ -H "Authorization: Bearer dummy_token" \ -H "Content-Type: application/json" \ -d '{ "patient_id": "P001", "consent_type": "genetic_screening", "consent_given": true }'

üîß Troubleshooting

Common Issues

Server Won't Start

Issue: Port 8001 is already in use
Solution: Kill existing processes or use a different port

Face Detection Fails

Issue: No face detected in image
Solution: Ensure image has clear, well-lit face with minimal background

Database Connection Error

Issue: Unable to open database file
Solution: Check file permissions and ensure data directory exists
